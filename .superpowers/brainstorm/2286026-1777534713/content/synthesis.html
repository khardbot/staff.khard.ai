Convergências que emergem da pesquisa
⚡ DESIGN.md é o protocolo de convergência
Neuform criou o conceito, Stitch 2.0 o adotou nativamente, e Pencil usa o mesmo princípio com .pen files. Os 3 compartilham a ideia de um "contexto de design portátil" que alimenta a AI em qualquer gerador.
Neuform Stitch 2.0 Pencil
🔗 MCP conecta tudo ao Claude Code (você)
Stitch tem MCP server para IDEs. Pencil é MCP bidirecional nativo com Claude. Isso significa que eu (Claude Code) posso ler o canvas do Pencil E escrever de volta — e consumir o design do Stitch — tudo via protocolo padrão.
Stitch MCP Pencil MCP bidirecional Claude Code
🎨 Onlook = padrão arquitetural, não dependência
Mesmo sem suportar SwiftUI, a arquitetura do Onlook (instrumentação em build-time + sourcemaps DOM→código + AST write-back) pode ser replicada para SwiftUI com Xcode Previews + Swift macros.
Conceito Onlook Xcode Previews Swift Macros
🚀 YC está apostando nesse espaço
Onlook (YC W25) e Pencil (YC-backed) foram acelerados no mesmo ciclo. Esse é o campo mais quente de tooling de AI para frontend. O timing para construir um workflow proprietário nesse espaço é agora.

Qual caminho tomar?
01
Web-first (PWA + Next.js)
Usa todas as 4 ferramentas nativamente. O app de residência roda como PWA instalável no celular. Iteração ultra-rápida sem App Store.
Stack
Neuform → DESIGN.md do produto
Stitch → Geração de telas por imagem
Pencil → Canvas + código no IDE
Onlook → Edição visual no Next.js
Claude Code ↔ MCP Pencil
✓ Recomendado para velocidade máxima
02
iOS-native (SwiftUI)
Usa Neuform + Stitch como referência visual. Pencil gera o protótipo web. Claude Code traduz para SwiftUI. Onlook como padrão conceitual via Xcode Previews.
Stack
Neuform → DESIGN.md (tokens e regras)
Stitch → Geração de mockup de referência
Pencil → Protótipo web validado
Claude Code → SwiftUI generation
Xcode Previews → "Onlook nativo"
⚠ Loop de iteração mais lento
03
Híbrido (Web valida → iOS produz)
Web valida UX com as 4 ferramentas em horas. Telas aprovadas são implementadas em SwiftUI. DESIGN.md compartilhado entre os dois ambientes.
Stack
Neuform DESIGN.md → fonte única de tokens
Stitch + Pencil → validação web rápida
Onlook → polish visual no web
Claude Code → SwiftUI das telas aprovadas
Design system bi-plataforma
⚠ Mais esforço · Maior qualidade final
